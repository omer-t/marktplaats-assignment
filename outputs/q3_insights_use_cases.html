Assignment¶

On insights and use cases:

  • What first impressions do you get from this data?
  • Name three use cases for this dataset (assume you might also combine it with other public or Adevinta datasets).

1. How This Continues Question 2¶

Question 2 treated the same car-ad data as an A/B testing problem: define the lead KPI, compare A and B, and decide how cautiously to interpret the result. Question 3 is broader. It asks what this dataset says about marketplace behavior and how the business could use it beyond one experiment.

I reuse the same lead definition from q2:

has_any_lead = telclicks > 0 OR bids > 0 OR n_asq > 0 OR webclicks > 0

For this question, I use all available car-ad rows with lead fields, not only rows assigned to A/B groups. That makes the first-impression analysis about the marketplace data itself, while the q2 results remain useful context for test-related caveats.

2. First Impressions¶

The dataset is rich at ad level: it combines car characteristics, listing characteristics, lead outcomes, and experiment assignment. That makes it useful for both performance analysis and business/product ideation, but it is not a full funnel dataset because it does not show buyer identity, seller history, transaction outcomes, revenue, or final sale price.

5. Segment Patterns¶

Segment views help turn raw columns into business language. Here I focus on price, mileage, car age, photo count, body type, and brand because they are understandable to both marketplace teams and sellers.

def segment_insight_summary(dataframe, segment_column, min_ads=500):
    summary = (
        dataframe.dropna(subset=[segment_column])
        .groupby(segment_column, observed=True)
        .agg(
            ads=("src_ad_id", "count"),
            lead_rate=("has_any_lead", "mean"),
            avg_total_leads=("total_leads", "mean"),
            median_price=("price", "median"),
            median_photos=("photo_cnt", "median"),
        )
        .reset_index()
    )
    summary = summary.loc[summary["ads"] >= min_ads].copy()
    summary["share_of_ads"] = summary["ads"] / len(dataframe)
    return summary.sort_values("lead_rate", ascending=False)

7. Recommendation¶

I would keep q2 and q3 as separate notebooks. Q2 is a test-design and A/B readout; q3 is a business-insight and use-case story. Combining them would make the notebook too broad and blur the difference between causal experiment analysis and descriptive marketplace exploration.

The clean portfolio structure is:

  1. q2_ab_test.ipynb: experiment setup, lead KPI, A/B result, risks, and recommendation.
  2. q3_insights_use_cases.ipynb: first impressions, reusable q2 findings, exploratory segment insights, and business use cases.

Final takeaway: this data is strong for understanding and improving lead generation. It should be paired with seller, buyer, revenue, and conversion data before using it for ROI decisions, pricing policy, or causal product claims.
